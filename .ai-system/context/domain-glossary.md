# 📖 Domain Glossary (Ubiquitous Language)
พจนานุกรมคำศัพท์เฉพาะของโปรเจกต์ เพื่อให้ทุกโมดูล (และ AI) ตั้งชื่อตัวแปร/คลาส ตรงกัน

## กฎการตั้งชื่อ (Naming Dictionary):
> ใส่คำศัพท์ที่ตกลงกันไว้ที่นี่ เพื่อป้องกันการใช้คำพ้องความหมายที่ทำให้เกิด Spaghetti Code

| คำศัพท์ที่ต้องใช้ (Approved Term) | ความหมาย / บริบท | ห้ามใช้คำเหล่านี้ (Banned Words) |
| :--- | :--- | :--- |
| `User` | ผู้ใช้งานที่ล็อกอินเข้าระบบ | Client, Account, Member |
| `Department` | หน่วยงานต้นสังกัดของผู้ใช้งาน | Org, Agency, Office |
| `Transaction` | รายการเคลื่อนไหวทางการเงิน/ระบบ | Record, Entry, Log |

## ทำไมต้องมีสิ่งนี้?
ในโปรเจกต์ขนาดใหญ่ หากคนหนึ่งเขียนโมดูลแรกว่า `clientId` แต่อีกคนเขียนโมดูลสองว่า `accountId` จะทำให้การเชื่อมข้อมูล (Join Table) พังพินาศ AI ต้องระวังเรื่องนี้และยึดคำศัพท์ในตารางนี้เป็นหลักสมอ
